DUNJOURN 0.0.1

OUTLINE

Here's a rough plan for what I want to implement, in three rough phases.

Things within a phase may not happen in the listed order, but all things in a phase will probably need to be done before I move on to the next one.

I'll also mention some less certain features I am considering that are not currently planned.

PHASE ONE:

  • more or less, finish a static version of this site with one "social dungeon": mine
  • some attempts will be made to keep in mind future plans, e.g. creature and item types will be read from an external file rather than be built-in classes of object the way they were in Flamecant, to make it easier to import them from user-provided files later
  • basic traits for items and creatures (e.g. "consumable", "weapon", "has_hands", etc)
  • obvi, multifloor dungeons
  • basic environmental interaction items (keys, torches, etc)
  • basic map tiles: floor and wall are done, but right now you can't use levelmaps that have stairs, doors, etc. shouldn't be too hard to add doors, stairs, some light source tile, maybe a water tile, etc.
  • improve inventory system from Flamecant (which was letter hotkeys only) to allow for easier inventory management

PHASE TWO:

  • the biggest phase, probably, this is when I will implement users/login, personal dungeons, etc
  • obvi that means a server and database, so I'll have to remember how to set that stuff up (I can probably manage it)
  • users, login, probably password recovery if I let people register with their emails
  • ability to upload a json with creature and item types, texts, etc
  • some ability to upload dungeons (json? some kind of text file? what's the simplest thing that I wouldn't have to build a level editor for?)

PHASE THREE:

  • this is when I get fancy with it
  • in-browser level editor!
  • text entry for passwords to secret rooms!

  • dim light, rendering darker glyphs and unhelpful tooltips?
  • various floor tiles rendered with different characters?
  • mouse controls?
